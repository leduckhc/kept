## UX Philosophy (from "How to use Superhuman like a semi-pro")

### Inbox Zero as Core Model
- Superhuman REQUIRES inbox zero management
- Archive = "done processing" (not delete)
- Inbox functions as your todo list
- "Get Me To Zero" feature: Cmd+K → type "zero" → archives all older emails

### Split Inboxes (Signature Feature)
- Create separate inboxes using Gmail filters
- Reduces context switching when triaging 250+ morning messages
- Important emails don't get buried
- Tips:
  - "Inbox" default split = emails NOT in another split
  - Keep Important/Other toggle OFF
  - Create an "All" inbox: search `in:inbox` → Cmd+K → create split inbox
  - Process splits in morning, use "All" during day

### Key UX Patterns
1. **Command Palette (Cmd+K)** — THE central interaction. Everything accessible from here.
2. **Keyboard-first** — mouse is optional, everything has a key binding
3. **Speed as feature** — every interaction < 100ms
4. **Visual density** — tight rows, monochrome, no wasted space
5. **Undo everything** — Z undoes any action with 5s window
6. **Split navigation** — Tab/Shift+Tab moves between inbox splits
7. **Inline comments** — team collaboration without forwarding
8. **Snippets (⌘+;)** — saved text blocks for repetitive replies
9. **Send Later (⌘+Shift+L)** — schedule emails
10. **AI search (Shift+/)** — natural language search
