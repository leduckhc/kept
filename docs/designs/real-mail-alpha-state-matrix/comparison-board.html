Recommended pattern across all variants

Two-line state copy, one primary action, one trust line

Recommend a consistent microcopy contract rather than picking the winning mockup shell. Every state should use: (1) a plain-language outcome line, (2) one trust/explanation line naming what stays local or what is missing, and (3) one dominant next step with optional quiet fallback.

Why this patternKeeps the no-sidebar inbox calm, matches existing Gmail Alpha empty/search copy, and prevents implementation from inventing separate voice systems for Gmail, search, reader, and AI.
Rationale bullets1. Trust copy stays visible exactly when data boundaries change. 2. The user always gets one obvious recovery step. 3. Technical detail moves behind secondary disclosure instead of polluting the main list.
