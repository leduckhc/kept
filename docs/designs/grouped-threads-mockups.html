Decision notes
A — Count badge: Simplest change. Removes the stacked-behind div, adds iOS-style badge. Works for both sender groups and domain groups identically.
B — Favicon strip: Best for domain groups where multiple distinct senders exist. More complex, may be overkill for single-sender groups.
C — Rounded square + metadata: Biggest departure. Signals "this is a group/bucket, not a person." Cleanest at scale but different visual language from individual thread rows.

Recommendation: A for sender-groups (same person, multiple threads), C for domain-groups (bucket of senders). Hybrid approach keeps individual threads round, groups square — instant visual differentiation.
